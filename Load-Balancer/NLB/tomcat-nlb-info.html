Understanding AWS Network Load Balancer (NLB)

AWS NLB operates at the fourth layer of the OSI model (TCP/UDP/TLS). It is optimized for routing ultra-high volumes of network-level traffic with minimal overhead.

Ultra-Low Latency

Operating at Layer 4, the NLB doesn't unpack the application headers. It routes packets directly at the transport layer, resulting in sub-millisecond latencies and incredible throughput.

Static & Elastic IPs

Unlike Application Load Balancers, an NLB reserves a single, persistent Static IP address for each Availability Zone. You can also assign Elastic IPs, making firewall whitelisting easy.

Millions of RPS

Designed to handle sudden, volatile traffic spikes. NLB can scale instantly from thousands of requests to millions of requests per second (RPS) without pre-warming.

NLB vs. ALB: Choosing the Right Balancer
Feature Network Load Balancer (NLB) Application Load Balancer (ALB)
OSI Layer Layer 4 (TCP, UDP, TLS) Layer 7 (HTTP, HTTPS, HTTP/2, gRPC)
Performance Ultra-high speed, microsecond latency High performance, millisecond latency
Routing Mechanism Port or Protocol-based routing Host header, Path, Query string, or Method routing
Static IP Support Yes (Static & Elastic IPs per AZ) No (DNS name only, dynamic IPs)
Health Checks TCP / HTTP / HTTPS active checks Deep HTTP/HTTPS content checks
